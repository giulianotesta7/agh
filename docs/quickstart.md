# Quickstart

Start AGH with Docker, log in with the first owner token, then run `agh pull` from a repo.

## 1. Start the server

Build the image:

```bash
docker build -t agh .
```

Run AGH with persistent state under `/data`:

```bash
docker run --rm -p 8912:8912 -v agh-data:/data \
  -e AGH_BOOTSTRAP_OWNER_EMAIL=owner@example.com \
  agh
```

Check the server:

```bash
curl http://127.0.0.1:8912/api/v1/health
```

## 2. Read the first owner token

AGH writes the token once:

```text
/data/secrets/initial_owner_token
```

Read it from the Docker volume:

```bash
docker run --rm -v agh-data:/data busybox \
  cat /data/secrets/initial_owner_token
```

## 3. Install the local CLI

From the AGH checkout:

```bash
./scripts/install.sh
```

Verify it:

```bash
agh --help
```

## 4. Log in

```bash
agh login \
  --url http://127.0.0.1:8912 \
  --email owner@example.com \
  --token "$(docker run --rm -v agh-data:/data busybox cat /data/secrets/initial_owner_token)"
```

Check the saved config. AGH masks the token:

```bash
agh config show
```

## 5. Create a project

```bash
agh project create \
  --repo-url "<your-git-remote-url>" \
  "<project-name>"
```

## 6. Link a repo

Run this inside the repo whose remote matches the project:

```bash
agh sync
```

AGH writes:

```text
.agh/project.toml
```

## 7. Pull the assigned packs

Preview first:

```bash
agh pull --dry-run
```

Apply them:

```bash
agh pull
```

Check local agent paths:

```bash
agh agent
```

## Files to commit

After a successful pull, commit the stable project state:

- `.agh/project.toml`
- `.agh/lock.toml`
- `AGENTS.md` / `CLAUDE.md` when present

Ignore the cache:

```gitignore
.agh-cache/
```

Skill targets under `.claude/skills/` and `.opencode/skills/` are generated by `agh pull`. Commit them only if your team wants those files reviewed in Git. If they are symlinks, run `agh pull` after clone to refresh `.agh-cache/packs/`.
