# Quickstart

Start AGH with Docker, log in with the first owner token, then apply assigned packs from a repo.

## 1. Start the server

Run AGH with the published image and persistent state under `/data`:

```bash
docker compose up -d
```

The compose file defaults to the latest published image:

```text
ghcr.io/giulianotesta7/agent-guidance-hub:${AGH_IMAGE_TAG:-latest}
```

Check the server:

```bash
curl http://127.0.0.1:8912/api/v1/health
```

## 2. Read the first owner token

AGH writes the token once:

```text
/data/secrets/initial_owner_token
```

Read it from the Docker volume:

```bash
docker run --rm -v agh-data:/data busybox \
  cat /data/secrets/initial_owner_token
```

## 3. Install the local CLI

From the published package:

```bash
uv tool install --force agh
```

Verify it:

```bash
agh --help
```

## 4. Log in

```bash
agh login \
  --url http://127.0.0.1:8912 \
  --email owner@example.com \
  --token "$(docker run --rm -v agh-data:/data busybox cat /data/secrets/initial_owner_token)"
```

Check the saved config. AGH masks the token:

```bash
agh config show
```

## 5. Create a project

```bash
agh project create \
  --repo-url "<your-git-remote-url>" \
  "<project-name>"
```

## 6. Link a repo

Run this inside the repo whose remote matches the project:

```bash
agh sync
```

AGH writes:

```text
.agh/project.toml
```

## 7. Select your local agent and pull packs

Each developer chooses one local agent target per workspace. The selection is stored in `.agh-cache/preferences.toml`, which is local cache state and should not be committed.

Set it explicitly:

```bash
agh agent select opencode
```

Or run pull in an interactive terminal and choose Claude Code, OpenCode, or Skip for now when prompted. Preview first:

```bash
agh pull --dry-run
```

Apply them:

```bash
agh pull
```

Check local agent paths and the current selection:

```bash
agh agent
agh agent show
```

## Files to commit

After a successful pull, commit the stable project state:

- `.agh/project.toml`
- `.agh/lock.toml`
- `AGENTS.md` / `CLAUDE.md` when present

Ignore the cache:

```gitignore
.agh-cache/
```

Skill targets under `.claude/skills/` or `.opencode/skills/` are generated by the workspace pull flow for the selected local agent. Commit them only if your team wants those files reviewed in Git. If they are symlinks, refresh the workspace after clone to rebuild `.agh-cache/packs/`.
